# Nitesh Kadve — Portfolio

Personal portfolio for Nitesh Kadve — Node.js Backend Developer focused on REST
APIs, real-time systems, and IoT. Built with **Vite + React + TypeScript + Tailwind CSS**.
Dark by default, navy/black/white theme, recruiter-oriented.

## Run

```bash
npm install
npm run dev       # http://localhost:3000
npm run build     # type-check + production build → dist/
npm run preview   # serve the production build locally
```

## Editing content

Almost all copy lives in [`src/lib/data.ts`](src/lib/data.ts) — profile, skills,
projects, architecture captions, timeline, and contact links. Edit there; the
components read from it.

Things to update before launch:

- **Links** in `src/lib/data.ts` → `profile.github`, `profile.linkedin`, `profile.website`.
- **Résumé** — replace `public/Nitesh-Kadve-Resume.pdf` with the real PDF.
  (The placeholder is regenerated by `node scripts/make-resume-pdf.mjs`.)
- **Domain / SEO** — the canonical URL, Open Graph, Twitter tags, and JSON-LD
  live statically in [`index.html`](index.html). Update the domain there and in
  [`public/robots.txt`](public/robots.txt) and [`public/sitemap.xml`](public/sitemap.xml).

## Structure

- `index.html` — app shell + all SEO/meta/Open Graph/Twitter tags + JSON-LD.
- `src/main.tsx` — entry point (fonts, global CSS, renders `App`).
- `src/App.tsx` — composes the page sections.
- `src/components/` — one component per homepage section.
- `src/lib/data.ts` — all site content.
- `src/globals.css` — Tailwind layers + custom CSS + font variables.
- `public/` — static files served as-is: `icon.svg`, `robots.txt`,
  `sitemap.xml`, `manifest.webmanifest`, résumé PDF.

## SEO

Metadata, Open Graph, Twitter cards, JSON-LD `Person` schema, canonical URL,
favicon, robots, sitemap, and manifest are all **static** — present in
`index.html` and `public/` so search engines and social scrapers (which don't
run JS) see them without executing the app. Semantic headings and a skip link
are in place for accessibility.

## Fonts

Inter and JetBrains Mono are self-hosted via `@fontsource-variable/*` (no
external requests, no layout shift). They are exposed to Tailwind through the
`--font-sans` / `--font-mono` CSS variables defined in `src/globals.css`.
